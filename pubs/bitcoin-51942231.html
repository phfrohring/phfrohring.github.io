Necessary properties

  • Countable: If we could not count money then exchange ratios would be meaningless: money must be countable.
  • Divisible: If we could not divide money then exchange ratios would be obviously impractical.
  • Portable: Imagine a dollar was made of two by two billboards‚Ä¶
  • Unforgeable: If someone could produce money at will w/o matching value then it would be equivalent to a theft: this guy could claim goods he did not produce matching value for. One could buy countries with a enough printing presses.
  • Durable: Who wants dollar bills that evaporate after a few hours?
  • Uniform: If a dollar in Texas was different from a follar in New York then a lot of confusion would obviously ensue.
  • Fungible: The value of a 1 dollar bill should always be equal to the value of an other 1 dollar bill whomever has possession of it.
  • Store Of Value: If it does not store value, how could it be useful besides for speculation?
  • Maybe others: Please, let me know if I missed something.

Double Spending Attacks

Double Spending Attacks

The risk that a digital currency can be spent twice. Double-spending is a problem unique to digital currencies because digital information can be reproduced relatively easily. You cannot double spend a 10$ bill.

Bizantine Generals Problem

Bizantine Generals Problem1

Reliable computer systems must handle malfunctioning components that give conflicting information to different parts of the system. This situation can be expressed abstractly in terms of a group of generals of the Byzantine army camped with their troops around an enemy city. Communicating only by messenger, the generals must agree upon a common battle plan. However, one or more of them may be traitors who will try to confuse the others. The problem is to find an algorithm to ensure that the loyal generals will reach agreement. It is shown that, using only oral messages, this problem is solvable if and only if more than two-thirds of the generals are loyal; so a single traitor can confound two loyal generals. With unforgeable written messages, the problem is solvable for any number of generals and possible traitors. Applications of the solutions to reliable computer systems are then discussed.

Green addresses

  • How to pay B if he does not have access to the Blockchain?
  • Directly, you cannot: there is no address to send BTC to.
  • You may send BTC to a third party (e.g.¬†a bank) who does have an address and the bank can tell B that he has been paid.
  • Third party guarantees agains double spend attack (thus: green address) else goes bust. Instawallet and Mount Gox went bust exactly this way.
  • Now, people are nervous about green addresses.

BitCoinNetwork

  • Runs on TCP, port 8333
  • Ad-hoc network, random topology
  • All nodes are equal, software wise.
  • Dynamic: new node can join/leave anytime
  • Inactive nodes for > 3Hr are forgotten
  • Incentive: Block reward
  • Incentive: Tx fee
  • HashPuzzle ‚â° solve hash256(nounce prev-hash tx ‚Ä¶ tx) ‚àà target-space for nounce. target-space is chosen by the network so that the average time to find a nounce is ~ 10 min.
  • Proof-Of-Work ‚â° Select node in proportion to its computing power i.e.¬†its ability to solve hash puzzles. Trivial for anyone to verify that puzzle was solved: check that hash256(block.nounce ‚Ä¶) ‚àà target.
  • Randomly choosen node to suggest a block to the network is approximated by Proof-Of-Work
  • Round ‚â° a node is randomly chosen and suggests a block
  • Check if block is valid (unspent, valid tx)
  • Nodes either prolong their concensus chain with this block or a block from an earlier round because of incentives.
  • Network properties:
    • As long as a majority of miners weighted by computing power are honest, BitCoin is secure.
    • It‚Äôs a currency: it includes incentives for nodes to act honestly (by the rules).
    • Nodes reach consensus upon one {Block,Block,‚Ä¶} ‚â° Blockchain
    • Works despite disconnected Nodes
    • Works despite crashed Nodes
    • Works despite malicious Nodes
    • Works despite network latency
    • Works despite Byzentine general problem
    • Works despite FischerLynchPaterson
    • Works despite many others a priori impossibilities
    • Nodes w/o identity ‚Üí protect from Sybil Attacks
    • No starting point for the consensus: reached in 1H, not matter where it originated from ‚Üí Never sure but wrong consensus is exponentially less likely to happen.

Propagating Tx

  • If given a new Tx then tell to as many peers as you can.
  • They can decide to do the same thing.
  • Decision is arbitrary but some sensible default:
    • Have I already propagated it?
    • Is the script standard?
    • Is the Tx valid?
    • Is it a double spend?

Propagating Blocks

  • If given a new Tx then tell to as many peers as you can.
  • They can decide to do the same thing.
  • Decision is arbitrary but some sensible default:
    • HashPuzzle solved?
    • All Tx valid?

Upgrading BitCoin
