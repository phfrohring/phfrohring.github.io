Deep Learning

Modifying a perceptron network so that a computer can find good enough W_1,W_2,\ldots w.r.t. a loss function itself is called Deep Learning.

Perceptron(\varphi) â‰¡ perceptron network where the Heaviside function has been replaced by a continuous differentiable function \varphi. Perceptron(\varphi) Loss Optimiser â†’ Deep Learning

If \varphi was not practically differentiable the backpropagation would not be possible.

Finding The Matrices

Given a Deep Learning Network, loss function E \equiv \frac{1}{2}(y - \hat{y})^2 and a gradient descent optimiser with a learning rate \eta:â„š, the matrices coefficients w_{i,k,j} are found by updating the coefficient like so:

\Delta w_{i,k,j} \equiv - \eta \frac{\partial E}{\partial w_{i,k,j}} w'_{i,k,j} â‰” w_{i,k,j} + \Delta w_{i,k,j}

Backpropagation

How to compute \frac{\partial E}{\partial w_{i,k,j}} ?

Once an output \hat{y} has been computed by the output node \text{node}_{n,1}, we have:

\frac{\partial E}{\partial w_{i,n,1}} \equiv \frac{\partial \hat{y}}{\partial w_{i,n,1}} \frac{\partial E}{\partial \hat{y}} Â Â Â Â \frac{\partial E}{\partial \hat{y}} \equiv - (y-\hat{y}) Â Â Â Â \hat{y} \equiv \varphi(\underline{\text{out}_{n-1}}\times{}W_{n,1}) Â Â Â Â \frac{\partial \hat{y}}{\partial w_{i,n,1}} \equiv \varphi'(\underline{\text{out}_{n-1}}\times{}W_{n,1})W_{i,n,1}

The last node can compute its weights: w'_{i,n,1} â‰” w_{i,n,1} - \eta \frac{\partial E}{\partial w_{i,n,1}}

Because the last node knows how much its output changes w.r.t. its inputs and how much the error changes w.r.t. its output, it can compute how much the error changes w.r.t. its inputs:

\frac{\partial E}{\partial \text{out}_{n-1,i}} \equiv \frac{\partial \hat{y}}{\partial \text{out}_{n-1,i}} \frac{\partial E}{\partial \hat{y}} Â Â Â Â \hat{y} \equiv \varphi(\underline{\text{out}_{n-1}}\times{}W_n) Â Â Â Â \frac{\partial \hat{y}}{\partial \text{out}_{n-1,i}} \equiv \varphi'(\underline{\text{out}_{n-1}}\times{}W_{n,1})W_{i,n,1}

Then, we back propagate \frac{\partial{} E}{\partial \text{out}_{n-1,i}} so that the \text{node}_{n-1,i} can update its weights:

\frac{\partial E}{\partial w_{r,n-1,i}} \equiv \frac{\partial \text{out}_{n-1,i}}{\partial w_{r,n-1,i}} \frac{\partial E}{\partial \text{out}_{n-1,i}}

w'_{r,n-1,i} â‰” w_{r,n-1,i} - \eta \frac{\partial E}{\partial w_{r,n-1,i}}

This recurrence in called Backpropagation.

To backpropagate from a k nodes layer to a n nodes layers, it suffices to use the multivariable chain rule: just sum the terms to compute \frac{\partial E}{\partial \text{out}_{n-1,i}}

Optimisation

Deep And Wide

In practice, deeper networks do better than wider network for a same amount of parameters, usually.

Larger networks then prevent overfitting more than smaller network: a little too small â‡’ will never fit.

Cross validation â‰¡ Measuring How Well It Generalizes

Split the data in training, validation and test sets. You may update the network wrt validation set but never the test set. The test set should be ideally used once.

How to measure effectiveness of updates you make on the network? Rule of thumbs: if more than 30 examples classified successfully a posteriori then significant.

Size of the validation test set?

Rule of thumbs: more than 30000 because a change of 30 means first decimal change i.e.Â good resolution.

Data scarcity may be mitigated by cross validation but more data is almost always better.

Stochastic Gradient Descent

Ideally, all data would be forwarded to the network at once. In practice, too much data. Learning happens in multiple steps: data is randomly sampled into batches, learned and again until data exhaustion and/or some requirements met.

rule of thumbs: If loss costs N floating points operations then its gradient costs about 3N.

Compute the loss function for a tiny amount of random data e.g.Â 1 to 1000 and use gradient descent on it. This si stochastic gradient descent.

It scales much better than gradient descent.

Batching

If the dataset exceeds available memory, e.g.Â the frame buffer of the GTX 1080 Ti is about 11 GB piece and some datasets reach terrabytes, then cut the dataset in random pieces and throw them at whatever memory you have.

Dicing and throwing random prieces of data is called batching.

Caveats

Numerical stability

Computers have limited amount of memory and computation power: too many operations and/or memory usage and it becomes useless â‡’ A computation should respect its computer

In particular, adding small things to big things lead to errors:

a = 1000000000
for i in range(1000000):
    a = a + 1e-6
print(a - 1000000000)
# prints 0.953674316406 instead of 1.0

Rule of thumbs: variables with zero mean and equal variances. Also, it let the optimiser have an idea of where the data is in the search space.

Minimizing Loss

Loss is a function of the weights: Loss â‰¡ Loss(W). The weights can be chosen to minimize Loss. How to initialize W?

Rule of thumbs: draw the loss from a zero centered gaussian distribution with small sigma.

Vanishing gradients on sigmoids

See: this post.

E.g., the sigmoid will be more useful on a 0 centered set with a reasonable spread because the sigmoid is linear around 0 but not far from it. Far from zero, â€œeverything looks the sameâ€ i.e.Â the derivative is close to 0.

Dying ReLUs

See: this post.

The forward and backward pass for a fully connected layer that uses ReLU would at the core include:

z = np.maximum(0, np.dot(W, x)) # forward pass
dW = np.outer(z > 0, x) # backward pass: local gradient for W

if a neuron gets clamped to zero in the forward pass (i.e.Â z=0, it doesnâ€™t â€œfireâ€), then its weights will get zero gradient.

Exploding gradients in RNNs

See: this post.

The gradient signal going backwards in time through all the hidden states is always being multiplied by the same matrix.

What happens when you take one number a and start multiplying it by some other number b (i.e.Â abbbbbbâ€¦)? This sequence either goes to zero if |b| < 1, or explodes to infinity when |b|>1. The same thing happens in the backward pass of an RNN, except b is a matrix and not just a number, so we have to reason about its largest eigenvalue instead.

Example with optimisation:


from tensorflow.examples.tutorials.mnist import input_data
import tensorflow as tf
import numpy as np
from helper import batches

# Load the data

mnist = input_data.read_data_sets('/datasets/ud730/mnist', one_hot=True)
n_input = 784   # img shape: 28*28
n_classes = 10  # 0-9 digits


## The features are already scaled and the data is shuffled

train_features = mnist.train.images
train_labels = mnist.train.labels.astype(np.float32)

valid_features = mnist.validation.images
valid_labels = mnist.validation.labels.astype(np.float32)

test_features = mnist.test.images
test_labels = mnist.test.labels.astype(np.float32)

# Define the network

## G(features,labels,learning_rate)

features = tf.placeholder(tf.float32, [None, n_input])
labels = tf.placeholder(tf.float32, [None, n_classes])

weights = tf.Variable(tf.random_normal([n_input, n_classes]))
bias = tf.Variable(tf.random_normal([n_classes]))
logits = tf.add(tf.matmul(features, weights), bias)


## Optimisation

learning_rate = tf.placeholder(tf.float32)
cost = tf.reduce_mean(tf.nn.softmax_cross_entropy_with_logits(logits=logits, labels=labels))
optimizer = tf.train.GradientDescentOptimizer(learning_rate=learning_rate).minimize(cost)


## Measure accuracy

correct_prediction = tf.equal(tf.argmax(logits, 1), tf.argmax(labels, 1))
accuracy = tf.reduce_mean(tf.cast(correct_prediction, tf.float32))


## Training parameters

batch_size = 128
epochs = 10
learn_rate = 0.001

# Compute

with tf.Session() as sess:
    sess.run(tf.global_variables_initializer())

    train_batches = batches(batch_size, train_features, train_labels)

    for epoch_i in range(epochs):

        for batch_features, batch_labels in train_batches:
            train_feed_dict = {
                features: batch_features,
                labels: batch_labels,
                learning_rate: learn_rate
            }
            sess.run(optimizer, feed_dict=train_feed_dict)

    test_accuracy = sess.run(
        accuracy,
        feed_dict={features: test_features, labels: test_labels})

print('Test Accuracy: {}'.format(test_accuracy))
